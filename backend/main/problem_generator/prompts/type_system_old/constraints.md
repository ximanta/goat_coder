# Type System Guidelines

1. Basic Types
   - Use primitive types when null is not possible
   - Use wrapper types when null is possible
   - Arrays must specify their element type

2. Cross-Language Compatibility
   - Avoid language-specific features
   - Use common type patterns
   - Follow nullable type conventions

3. Function Signatures
   - Maximum 2 parameters for easy problems
   - Single return type
   - Clear parameter names

4. Type Naming Conventions
   - Use PascalCase for class types
   - Use camelCase for variable names
   - Use clear type annotations
